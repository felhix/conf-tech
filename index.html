Qu'est-ce qu'un langage de programmation ?

Un langage de programmation permet de "parler" à son ordinateur et lui demander d'exécuter des tâches. Tout langage de programmation est ensuite compilé en binaire afin que l'ordinateur le comprenne.

Exemples de tâches : afficher "bonjour monde", créér un nouveau dossier, ajouter une adresse email à un utilisateur en base de données. Pourquoi binaire btw ? Pck à la base, un ordinateur est un circuit électronique amélioré, et qu'un circuit électronique ne connait que deux états : allumé, ou éteint. D'où 0 et 1. Enfin 2⁴ == 16

Exemple langages

Exemple de langages : C, C++, Java, Ruby, Python

HTML et CSS ne sont pas des langages de programmation

HTML est un langage de balisage et CSS un langage informatique. Ces deux derniers ne permettent pas de faire de la programmation

Front, Back, Framework ?

Jargons expliqués

Back-end

C'est le plus commun, ce sont les programmes qui permettent d'exécuter du code qui fera des opérations et algorithmes.

Exemples langages de back-end : C, C++, Java, Ruby, Python

On reviendra sur chaque langage.

Back-end

Quelle est la différence entre les langages de back-end ? Pourquoi défendre un langage, c'est comme défendre son équipe de foot préférée ?

Exemples de points de différenciations entre langages de programmations : Haut niveau, ou bas niveau ; Statique ou dynamique ; portabilité ? Orienté objet ?

Bas niveau : plomberie, mais maîtrise totale de ce que nous faisons. Statique : les variables ne peuvent être assignées qu'à un seul type de données : moins pratique, mais moins prône à l'erreur. Cela reste technique

Front-end

La partie visible de l'iceberg. Toutes les parties front de sites, jeux-vidéos, applications, sont codées avec des langage de Front-end, qui sont exécutés par le navigateur (Chrome, Windows, iPhone, etc)

Exemples langages front-end : HTML / CSS / JavaScript

Les langages sont exécutés par notre navigateur. C'est à dire qu'il nous envoie tout le code, et en échange affiche un joli site. C'est pour cela que l'on parle de problème de compatibilité, ou que l'on trashe IE6.

Différences entre langages

  • HTML : fait le squelette
  • CSS : fait la mise en forme du site
  • JavaScript : fait l'animation du site en fonction de ce que fait l'utilisateur
Exemple page web : si on prend le site du monde, tout le texte des articles est en HTML. Puis le CSS dit "tel article est là". Enfin, le JS est là pour par exemple nous rappeler à l'ordre si on utilise adblock, avec un truc du genre if adblock enabled, send popup.

Exemple : Comment marche un site

  • facebook.com est différent pour tout le monde
  • Quand un user arrive sur facebook.com, le site va chercher sur ses serveurs tous les posts de ses amis
  • Ensuite, le site va appliquer dans son back-end un algorithme qui va selon lui sortir les résultats qu'il estime les plus pertinents
  • Enfin, le back-end va envoyer ceci convertit en HTML/CSS/JS pour que le Chrome (ou autre) de l'utilisateur l'exécute
Si tu vas sur fb.com, tout le monde aura un truc différent. Quand tu arrives, le site va exécuter du code en backend, en cherchant dans sa base tous les posts. Il applique sur tous les posts un algo qui lui semble pertinent, puis les renvoie sous forme de html css js

La programmation web

Pour faire tourner les serveurs et donc le back-end d'un site, il faut des langages spécialisés pour faire le lien entre le back-end et le front-end.

Exemples langages web : Node.js, PHP, Rails, Django

On peut faire tourner un site en C, mais c'est mieux d'utiliser dans langages spécialisés car ces derniers mâchent le travail du développeur

Serveurs ?

Certains sites n'ont pas besoin de back-end, donc tout le code sera exécuté par le navigateur de l'utilisateur. On dit qu'ils sont statiques.

En revanche, dès qu'un site a un backend, il faut une machine pour faire tourner le code : c'est le rôle des serveurs. On dit qu'ils sont dynamiques

En général, quand toutes les url de tout un site renvoient à chaque fois la même page à tout le monde, ce site est statique (exemple : ta landing, qui sera la même pour tout le monde). En revanche, dès que tu dois afficher une page différente en fonction de paramètres prédéfinis, le site est dynamique. Exemple : facebook, gmail

Les datas ?

Quand on créé un utilisateur sur le site, ses informations (emails, adresse, etc) sont stockées dans des bases de données. Une BDD est grosso-modo une ribambelle de classeurs excels reliés entre eux. Facebook a par exemple une table utilisateurs, et une table messages. Un utilisateur peut avoir plusieurs messages, mais un message n'a qu'un seul expéditeur.

SQL est généralement utilisé pour la data.

Framework ? Library ?

Les frameworks sont en quelque sorte des extensions de langages qui mâchent le travail des développeurs.

Les libraries font ça, mais pour des problématiques plus localisées. Un framework pourra être considéré comme un langagze à part entière, tandis qu'une library non.

exemples langages et comment ils boostent le travail. Rails, Django, Bootstrap, jQuery. Library : Twitter, BeautifoulSoup

Listing des langages

À quoi ils servent ?

Assembleur

Le langage le plus bas-niveau qui soit. Assembleur est l'un des premiers langages créé. Très très peu de gens codent en Assembleur, et c'est vraiment un langage de barbus. Utilisé pour les hardware.

niveau d'après : binaire

C

Un langage très bas-niveau, très rapide à l'exécution, très lent à écrire. C est compilé, ce qui veut dire qu'il est converti

C++ / C# / Objective C

C++ / C# / Objective C sont à C ce que C est à l'Assembleur. Ce sont des langages versatiles, puissants pour faire tout type d'app. Windows est codé en C++, macOS en Objective C (avec un peu de C)

Java

Un langage extrêmement populaire, Java a l'avantage de pouvoir tourner sur quasimment n'importe quelle machine.

Python

Plus haut niveau, et plus lent que les langages ci-haut, Python se distingue pour sa simplicité, où les devs passeront plus de temps à se soucier de leurs algorithmes, que des troubles de programmation. C'est un lanage très populaire plesbiscité par la communauté des chercheurs (donc a plein de libraries de data science)

Django

Django est un framework qui permet d'ajouter une fonctionnalité très intéressante à Python : faire du web

Ruby

Ruby est très similaire à Python : un langage haut-niveau qui se distingue pour sa simplicité. Moins populaire que Python, il a l'avantage d'avoir un framewok très puissant (Rails), et de vouloir le bonheur du développeur.

Go

Go est un langage très récent qui est très bas-niveau, mais apporte une syntaxe moderne à des équivalents comme C.

Swift

Swift est un langage de programmation développé par Apple. Il sert à développer des apps iOS, macOS, etc

PHP

PHP est un langage spécialisé pour faire du web. Il était très hype vers 2004, mais maintenant c'est devenu une blague. Récemment, ses deux framework Symphony et Laravel ont bien grimpé en popularité.

Javascript

LE langage le plus hype du moment. Désigné en 10 jours, son omniprésence dans le web en a fait un langage indispensable dans la carrière d'un dev. Voici quelques frameworks :

  • jQuery : permet de manipuler les éléments d'une page web
  • Node : permet d'exécuter JavaScript en tant que langage de back-end
  • React-Native : permet de développer des applications hybrides iOS / Android
  • Angular : permet de faire des Single Page Application
